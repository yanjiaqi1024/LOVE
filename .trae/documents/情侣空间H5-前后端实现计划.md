## Summary
- 目标：实现一个可上线的情侣空间 H5 单页网站（移动端优先、浪漫粉/浅紫配色），并提供 FastAPI + SQLite 后端用于账号登录与恋爱资料/打卡/纪念日跨设备同步；相册图片本地用 IndexedDB 存储，后端同步相册“元数据”（可扩展为云相册）。

## Current State Analysis
- 仓库目录为空（未发现现有前端/后端工程文件），需从 0 脚手架项目结构与基础代码。
- 需求来源于提示词：UI 侧强调高颜值、移动端适配、交互动效、无报错；功能侧包含登录注册、首页在一起天数与打卡、相册上传预览删除、个人中心资料与纪念日倒计时。
- 决策已确认：
  - 交付方式：纯静态前端（Tailwind 走 CDN，无构建步骤）。
  - 账号模型：单账号（单对情侣）为主，但数据通过后端可跨设备同步。
  - 相册存储：IndexedDB（避免 localStorage 容量限制）。
  - 后端用途：账号 + 恋爱资料同步。
  - 后端栈：Python + FastAPI。
  - 数据库：SQLite。

## Proposed Architecture
### Frontend（纯静态 SPA）
- 技术栈：HTML + Tailwind CSS（CDN）+ 原生 JavaScript（模块化）+ localStorage（token/设置缓存）+ IndexedDB（相册图片）。
- 页面组织：单页多视图（Login / Home / Album / Me），通过 hash 路由（#login/#home/#album/#me）或基于状态的视图切换实现；页面切换使用 CSS transition 实现顺滑动效。
- 数据策略：
  - 在线：通过后端 API 读写账号资料、打卡、纪念日等；前端做轻量缓存与离线兜底。
  - 离线：仍可查看本地缓存；写操作排队（可选）或提示网络不可用。
  - 相册：图片二进制存 IndexedDB；相册列表/排序/封面等元数据存本地并可同步到后端（不强制上传图片）。

### Backend（FastAPI + SQLite）
- 功能边界：提供账号认证与“结构化数据”同步 API（资料/打卡/纪念日/相册元数据）。不强制承担图片存储；如需要云相册，可后续增加 /photos/upload 等接口与文件存储。
- 认证：JWT（access token）或 session cookie（二选一，计划默认 JWT 以便纯静态前端部署）。
- 数据库：SQLite（SQLModel/SQLAlchemy），单实例部署即可。

### 2) 后端 API 设计（实现阶段固定）
- POST /api/auth/register
  - body: { username, password }
  - 约束：单账号模式下，若已存在用户则返回 409
- POST /api/auth/login
  - body: { username, password }
  - resp: { access_token, token_type: "bearer" }
- GET /api/me
  - auth: bearer token
  - resp: { id, username, profile }
- GET/PUT /api/profile
  - profile: { yourNickname, partnerNickname, yourAvatar, partnerAvatar, metDate, loveDate, slogan }
- POST /api/checkins/today
  - 规则：每天仅一次；返回 { checkedInToday, streakDays, totalDays }
- GET /api/checkins/summary
  - 返回 { checkedInToday, streakDays, totalDays, lastCheckinDate }
- GET/POST /api/anniversaries
- PUT/DELETE /api/anniversaries/{id}
- GET/POST /api/album-meta
- PUT/DELETE /api/album-meta/{id}

### 3) 前端功能拆解（实现阶段固定）
- 登录注册页
  - 表单校验、错误提示、成功后保存 token 到 localStorage 并跳转首页
  - 未登录访问其他路由时，强制跳转 #login
- 首页
  - 顶部昵称与问候语（根据时间段生成）
  - 在一起天数：基于 loveDate 计算（按本地时区日历天），实时展示
  - 每日打卡：按钮动效（点击缩放/波纹），已打卡禁用；展示连续/累计天数
  - 恋爱语录：随机展示，可刷新切换
- 相册页
  - 上传图片：input file 选择后生成缩略图；写入 IndexedDB
  - 网格/瀑布流展示（移动端 2 列优先）
  - 全屏预览：遮罩层、左右滑动切换（touch events）、关闭
  - 删除：确认弹窗；删除 IndexedDB 记录；同步删除相册元数据（如启用）
- 我的页
  - 展示双方头像、昵称、在一起总天数
  - 恋爱资料编辑：日期选择器（原生 input[type=date]）
  - 纪念日：自定义名称+日期，列表显示倒计时；支持增删改

## Assumptions & Decisions
- Tailwind 使用 CDN：避免构建，适合快速上线静态页；少量自定义 CSS 放到 /frontend/css/styles.css。
- 账号单对情侣：后端限制单用户注册；跨设备同步依赖该账号登录。
- 相册默认“本地为主”：图片存 IndexedDB；后端仅同步元数据（标题/时间/排序/封面 id 等）。如你后续明确要云相册，再在计划基础上扩展图片上传与存储。
- 登录态：JWT 存 localStorage；所有 API 请求统一在 api.js 注入 Authorization 头。

## Verification Steps (Implementation Phase)
- 前端
  - 打开 /frontend/index.html：在移动端视口下检查布局、字体、配色与交互动效
  - 注册/登录：token 持久化，刷新后仍保持登录并正确路由
  - 打卡：同一天多次点击只允许一次；跨天后可再次打卡；连续/累计计算正确
  - 相册：上传多张、缩略图渲染、全屏预览滑动、删除生效；刷新后仍存在（IndexedDB）
  - 纪念日：新增/编辑/删除，倒计时正确（以本地日历天计算）
- 后端
  - 启动 FastAPI：OpenAPI 文档可用
  - Auth：注册一次成功；重复注册返回 409；登录失败返回 401
  - Profile/Checkin/Anniversary：CRUD 与规则校验通过；数据落盘到 SQLite
  - CORS：允许前端静态站点访问（开发期允许 localhost）
